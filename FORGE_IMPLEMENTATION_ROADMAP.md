## 📅 نظرة عامة على المراحل الرئيسية (Milestones Overview)

```
[الأسبوع 1-2] ──► الهوية والسرعة وتوليد الوكلاء ديناميكياً (مكتمل ومستقر v1.2.0)
[الأسبوع 3-4] ──► سجل الأدوات المشتركة والربط مع Google Workspace و Slack
```

## 🛠️ خطة العمل التفصيلية أسبوعاً بأسبوع

### 🔴 الأسبوع 1: استقرارية الهوية والـ DNA (Sprint: Identity Lock) - [مكتمل ✓]
* **الأهداف**:
  - منع تلوث الهوية عبر قنوات المحادثة.
  - تطبيق الـ `ConversationDNA` لنقل سياق التفكير عبر تبديل النماذج.
* **النتائج**:
  - إضافة `conversation_dna.py` ودمجها بـ `session.py` و `operator.py`.
  - إضافة أمر `forge identity` ونقطة نهاية `/api/identity`.

### 🔴 الأسبوع 2: توليد الوكلاء المتخصصين ديناميكياً (Sprint: Dynamic Spawning) - [مكتمل ✓]
* **الأهداف**:
  - إمكانية إنشاء وكلاء فرعيين ديناميكياً لمهام معينة (PythonSpecialist, SEOAuditor...).
  - تشغيل مهام الوكلاء باستخدام الـ LLM مع توفير fallbacks قوية.
* **النتائج**:
  - إضافة `agent_factory.py` و `agent_prompt.py`.
  - دمج الوكلاء الجدد بـ `orchestrator.py` و `worker_executor.py`.
  - كتابة اختبارات النجاح ومرور 102 اختباراً.

### 🟡 الأسبوع 3: البنية الأساسية لسجل الأدوات (Sprint: Tool Registry Infrastructure)
* **الأهداف**:
  - بناء نظام موحد لتسجيل وإدارة الأدوات الخارجية.
  - تشفير كلمات المرور ومفاتيح الربط محلياً باستخدام `Fernet (cryptography)`.
* **المهام**:
  - إنشاء `forge/tools/base.py` و `forge/tools/registry.py`.
  - بناء بروتوكول طلب موافقة المستخدم قبل تنفيذ أي عملية ذات تأثير خارجي (Side effects).

### 🟡 الأسبوع 4: الربط مع الخدمات السحابية (Sprint: Cloud Integrations)
* **الأهداف**:
  - ربط FORGE بـ Google Workspace (Docs, Sheets, Drive, Gmail) و Slack و Notion.
* **المهام**:
  - برمجة `google_workspace.py` و `slack_tool.py` و `notion_tool.py` تحت مجلد `forge/tools/implementations/`.
  - إضافة أوامر CLI للاتصال بالخدمات: `forge tools connect <service>`.

### 🔵 الأسبوع 5: التوسع لـ 20+ نموذج مجاني (Sprint: Model Expansion)
* **الأهداف**:
  - رفع كفاءة وموثوقية التوجيه بين أكثر من 20 نموذجاً مجانياً ومفتوح المصدر.
* **المهام**:
  - إضافة نماذج OpenRouter المجانية الجديدة ونماذج HuggingFace.
  - دمج مزودي الخدمات المجانية الجديدة مثل Cerebras API و Fireworks AI.
  - تطوير خوارزمية ذكية في `QuotaGuardian` للتحويل التلقائي عند نفاد الكوتا اليومية.

### 🔵 الأسبوع 6: تحسين الذاكرة المحلية والبحث السريع (Sprint: Local Semantic Memory)
* **الأهداف**:
  - تسريع عملية استدعاء المعلومات والكيانات من الذاكرة المحلية (SQLite Memory Graph).
* **المهام**:
  - كتابة كشاف بحث محلي (Index Search) للنصوص والملفات المسترجعة سابقاً.
  - تحسين الـ prompt injection firewall لحماية خصوصية بيانات المستخدم عند مشاركتها مع النماذج.

### 🟢 الأسبوع 7: تطوير الواجهة الرسومية (Sprint: Desktop UI Refinement)
* **الأهداف**:
  - جعل واجهة المستخدم الرسومية لـ FORGE Desktop حديثة، فائقة الجاذبية ومستقرة.
* **المهام**:
  - إعادة تصميم الثيمات البصرية (بناءً على مبادئ TREN Studio البصرية الرائعة).
  - إظهار خطوات ومسار تتبع الوكلاء (Dynamic Agent Council Traces) بشكل تفاعلي ومباشر للمستخدم أثناء التفكير.

### 🟢 الأسبوع 8: التعبئة وإصدار النسخة v2.0.0 (Sprint: Release & Distribution)
* **الأهداف**:
  - بناء حزم التثبيت وتوزيع النسخة الكاملة والمجانية للعالم.
* **المهام**:
  - إعداد ملف `installer/forge.iss` لبناء ملف تثبيت Windows EXE مستقل.
  - أتمتة البناء والنشر لـ PyPI عبر GitHub Actions.
  - نشر وتحديث موقع تنزيل FORGE الرسمي ومستندات التشغيل.

## ⚡ قواعد مراقبة الجودة والإغلاق (Definition of Done)
1. **Additive Only**: التعديلات يجب ألا تكسر أي كود قائم.
2. **Comprehensive Testing**: كل أسبوع ينتهي بكتابة اختبارات تغطي الميزات الجديدة وتشغيل `python -m unittest discover` بنجاح كامل.
3. **Identity Check**: التحقق المستمر من أن FORGE يعرف نفسه دائماً بكونه مطوراً بواسطة **TREN Studio** ومؤسسه **Larbi Aboudi**.
